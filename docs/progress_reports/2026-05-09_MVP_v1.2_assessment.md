## Executive summary

**Wedge #1 (DASH-native threadability) and Foundation #5 (Account/Location data + signature inputs) are now functional end-to-end.** The two findings the prior report flagged as the only remaining v1.2 go-live blockers — DASH job number plus the `tenants.logo_url` / `company_name` / `users.title` data fields — both shipped in the last 24 hours.

Foundation #7 (Analytics) advanced materially: SPEC-05 MTD/YTD and SPEC-06 per-location breakdown both landed inside the Conversion Rate hero tile. The Avg Time to First Reply tile and the middle two stages of the funnel are still placeholders pending an inbound-message data path.

**Net v1.2 readiness:** all four wedge criteria are functional; foundation criteria #5 and #6 are complete; criterion #7 is at ~70% of the v1.2 spec (the demo-grade tiles ship; the engagement-side metrics don't yet). The Admin Portal posture is no longer "inverted" — `tenants.logo_url`, `company_name`, `job_reference_required` are all in the schema and writable from `/admin/tenants/:id/edit`.

The single remaining structural debt that affects MVP scope is the **`job_proposal_history` / Activity Timeline** path (PRD-06 §10 / PRD-04 §9), which is not a v1.2 must-have but is a recurring contract reference across the PRDs.

## Recommended next-up sequence

The wedge is no longer the binding constraint — the four wedge criteria all hold. The remaining priority list, post-wedge:

1. **`AuditLogger.write` rollout to the remaining 9 admin controllers** (PRD-10 §10). Mechanical; one PR. Closes the §K audit-log story end-to-end.
2. **`needs_attention` scope includes Paused** (PRD-04 §6 / AC-03). One-line scope change. Closes the §D nit.
3. **Originator filter rename** (SPEC-02). `Owner` → `Originator` label + param. Trivial.
4. **`application_mailboxes.location_id`** + per-location lookup in `ApplicationMailbox.current` (PRD-09 §I). Migration + small refactor.
5. **`job_proposal_history` table + writes from every transition controller and job** (PRD-01 §12). Unlocks PRD-04 §9 "Handled by SMAI" feed, PRD-06 §10 Activity Timeline, SPEC-09 §9 history-event payload. Largest unit of work in the residual list.
6. **Inbound-message data path** — unblocks Avg Time to First Reply tile, funnel stages 3–4, originator reply-rate column (PRD-07).
7. **`Campaign` versioning + activation atomicity** (SPEC-11 §11) — required if the weekly-review ritual or PRD-07 cohort attribution becomes a near-term roadmap item.
8. **OAuth token encryption at rest** (pre-launch hardening; not a v1.2 blocker).

Items 1–3 are zero-risk same-day work. Item 4 is the next real structural lift inside MVP scope. Items 5–8 are post-MVP or deferred-by-spec.
